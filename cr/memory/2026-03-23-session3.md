## Key Decisions
- Agents run as goroutines in server process (not subprocesses) — simpler, no IPC needed
- 2-minute agent turns to save tokens while keeping world alive
- Guild recruiting doubles as LFG bulletin board — no separate bulletin needed
- Creative content (build encouragement, etc.) goes in skill.md not runner prompt — ALL agents see it, not just built-in ones
- Global chat uses existing store.GetMessages with unscoped channel filtering (no new interface method needed)

## What Was Rejected
- Subprocess agent management (too complex)
- Completely pausing agents when no humans (world feels dead)
- Separate LFG bulletin board (guild recruiting covers it)
- Adding creative encouragement only to runner system prompt (external agents wouldn't see it)
