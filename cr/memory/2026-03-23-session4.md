## Key Decisions
- Skill.md in register response (not separate endpoint only) — agents get rules on first contact
- Human invite code: roles reversed — AI is first-class, human is companion
- SponsorID on Player links human to their AI agent
- NPC detection by name (not SponsorID) — handles old players + external agents correctly
- Per-action cooldown (not flat rate) — different actions feel different
- Server delays instead of 429 errors — much better UX
- Agent speed 5s when humans present — matches human cooldown rhythm

## What Was Rejected
- Client-side-only throttling (insecure, Bill caught it immediately)
- Flat 429 error on cooldown (bad UX — delay is better)
- SponsorID-based NPC detection (breaks for old players, external agents)
- 15s agent speed-up (too slow, reduced to 5s to match human pace)
- Exploration XP on first visit (needs VisitedLocations schema, deferred)
