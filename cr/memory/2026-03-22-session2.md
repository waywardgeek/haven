## Key Architecture Decisions
- **Story replaces Quest entirely**: Quest struct was too basic, Story subsumes it completely
- **Instancing reuses existing systems**: instanced locations are real Location objects with CreatedBy=instance_id, normal movement/combat/loot works inside them
- **Completion detection via location property**: rooms with "story_completion" in Properties trigger auto-completion when entered
- **PR model for location edits**: non-owners suggest, owners approve — mirrors GitHub PR workflow
- **Ownership fallback at 500 ticks**: prevents abandoned locations from being uneditable

## What Was Considered and Rejected
- Separate adventure engine: rejected — reusing existing Location/Creature/Store is simpler and keeps one system
- WebSocket for story events: not needed — same polling works
- Complex party finding system: KISS — just party ID in start request
- DeleteLocation for instance cleanup: not in Store interface yet — instead clear exits and mark description as ended
